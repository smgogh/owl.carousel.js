概要

响应选项可用于设置断点和其它选项。试着改变你的浏览器宽度，以看到项目和导航发生什么。

关于响应选项

该响应的设置非常简单。响应选项结构：

responsive : {
    // breakpoint from 0 up
    0 : {
        option1 : value,
        option2 : value,
        ...
    },
    // breakpoint from 480 up
    480 : {
        option1 : value,
        option2 : value,
        ...
    },
    // breakpoint from 768 up
    768 : {
        option1 : value,
        option2 : value,
        ...
    }
}

关键事实：

  • 每个断点键可以是一个数字值（如例子），或者一个字符串：'480'.
  • 猫头鹰有一个内置的排序选项，但最好还是从最小屏到最宽屏进行设置。
  • 响应选项总是覆盖顶层设置。
  • 由于默认情况下，响应选项设置为true，所以旋转木马总是试图以适应包装（即使媒体查询不支持IE7 / IE8等）。
  • 如果没有灵活的布局可以设置 responsive:false.

举个栗子

$('.owl-carousel').owlCarousel({
    loop:true,
    margin:10,
    responsiveClass:true,
    responsive:{
        0:{
            items:1,
            nav:true
        },
        600:{
            items:3,
            nav:false
        },
        1000:{
            items:5,
            nav:true,
            loop:false
        }
    }
})

响应相关选项：

responsiveClass

可选的辅助类。 添加 'owl-reponsive-' + '断点' 类作为主要元素。

responsiveBaseElement

由于默认情况下所有响应断点与对应的 window 宽度。这个选项给你一个机会，将其更改为自己喜欢的类/ID responsiveBaseElement:".myCustomWrapper"

responsiveRefreshRate

这里做的事情就是在更改了浏览器的宽度并进行刷新操作（计算宽/克隆项目等）之后等待200ms，默认刷新率为200ms。我觉得这个速度最佳，如果你觉得它慢了，你可以改变它。
